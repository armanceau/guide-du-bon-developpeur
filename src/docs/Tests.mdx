# 🧪 Tests

Les tests sont essentiels pour garantir la qualité, la fiabilité et la maintenabilité de votre code.

## 🎯 Pourquoi tester ?

### Confiance

- Déployer sans peur
- Refactorer en toute sécurité
- Modifier le code existant sans casser

### Documentation

- Les tests servent de documentation vivante
- Montrent comment utiliser le code
- Explicitent les comportements attendus

### Qualité

- Détection précoce des bugs
- Force à écrire du code testable (donc mieux conçu)
- Prévention des régressions

## 📊 La pyramide des tests

```
        /\
       /E2E\      ← Peu nombreux, lents, coûteux
      /------\
     / Intég.\   ← Moyennement nombreux
    /----------\
   / Unitaires \  ← Très nombreux, rapides, peu coûteux
  /--------------\
```

### Tests unitaires (70%)

Testent une unité de code isolée (fonction, méthode, composant)

### Tests d'intégration (20%)

Testent l'interaction entre plusieurs unités

### Tests E2E (10%)

Testent l'application complète du point de vue utilisateur

## 🔬 Tests unitaires

### Principes

**AAA Pattern : Arrange, Act, Assert**

```javascript
test("additionne deux nombres", () => {
  // Arrange - Préparer les données
  const a = 5;
  const b = 3;

  // Act - Exécuter la fonction
  const result = add(a, b);

  // Assert - Vérifier le résultat
  expect(result).toBe(8);
});
```

### Bonnes pratiques

#### 1. Un test = un concept

```javascript
❌ Mauvais
test('user operations', () => {
  expect(createUser()).toBeTruthy();
  expect(deleteUser()).toBeTruthy();
  expect(updateUser()).toBeTruthy();
});

✅ Bon
test('creates a new user', () => {
  expect(createUser()).toBeTruthy();
});

test('deletes an existing user', () => {
  expect(deleteUser()).toBeTruthy();
});
```

#### 2. Noms de tests descriptifs

```javascript
❌ Mauvais
test('test1', () => { ... });
test('it works', () => { ... });

✅ Bon
test('returns null when user is not found', () => { ... });
test('throws error when email is invalid', () => { ... });
```

#### 3. Tests indépendants

```javascript
❌ Mauvais - Tests dépendants
let user;

test('creates user', () => {
  user = createUser();
  expect(user).toBeTruthy();
});

test('updates user', () => {
  user.name = 'New Name'; // Dépend du test précédent !
  expect(user.name).toBe('New Name');
});

✅ Bon - Tests indépendants
test('creates user', () => {
  const user = createUser();
  expect(user).toBeTruthy();
});

test('updates user name', () => {
  const user = createUser(); // Crée son propre user
  user.name = 'New Name';
  expect(user.name).toBe('New Name');
});
```

### Exemple complet

```javascript
// user.service.js
export class UserService {
  constructor(database) {
    this.db = database;
  }

  async createUser(email, password) {
    if (!email.includes("@")) {
      throw new Error("Invalid email");
    }

    const existingUser = await this.db.findByEmail(email);
    if (existingUser) {
      throw new Error("User already exists");
    }

    return this.db.create({ email, password });
  }
}

// user.service.test.js
import { UserService } from "./user.service";

describe("UserService", () => {
  let userService;
  let mockDatabase;

  beforeEach(() => {
    // Arrange - Mock de la base de données
    mockDatabase = {
      findByEmail: jest.fn(),
      create: jest.fn(),
    };
    userService = new UserService(mockDatabase);
  });

  describe("createUser", () => {
    test("creates user with valid email and password", async () => {
      // Arrange
      const email = "test@example.com";
      const password = "password123";
      mockDatabase.findByEmail.mockResolvedValue(null);
      mockDatabase.create.mockResolvedValue({ id: 1, email });

      // Act
      const result = await userService.createUser(email, password);

      // Assert
      expect(result).toEqual({ id: 1, email });
      expect(mockDatabase.create).toHaveBeenCalledWith({ email, password });
    });

    test("throws error when email is invalid", async () => {
      // Arrange & Act & Assert
      await expect(
        userService.createUser("invalid-email", "password"),
      ).rejects.toThrow("Invalid email");
    });

    test("throws error when user already exists", async () => {
      // Arrange
      mockDatabase.findByEmail.mockResolvedValue({ id: 1 });

      // Act & Assert
      await expect(
        userService.createUser("test@example.com", "password"),
      ).rejects.toThrow("User already exists");
    });
  });
});
```

## 🔗 Tests d'intégration

Testent plusieurs unités qui travaillent ensemble.

```javascript
// api.integration.test.js
describe("User API Integration", () => {
  let app;
  let database;

  beforeAll(async () => {
    // Setup real database for integration tests
    database = await setupTestDatabase();
    app = createApp(database);
  });

  afterAll(async () => {
    await database.close();
  });

  test("POST /users creates a new user", async () => {
    const response = await request(app)
      .post("/users")
      .send({ email: "test@example.com", password: "password123" });

    expect(response.status).toBe(201);
    expect(response.body).toHaveProperty("id");
    expect(response.body.email).toBe("test@example.com");

    // Verify in database
    const user = await database.findById(response.body.id);
    expect(user).toBeTruthy();
  });
});
```

## 🌐 Tests End-to-End (E2E)

Testent l'application complète dans un navigateur réel.

```javascript
// Playwright example
import { test, expect } from "@playwright/test";

test("user can login and view dashboard", async ({ page }) => {
  // Navigate to login page
  await page.goto("http://localhost:3000/login");

  // Fill login form
  await page.fill('input[name="email"]', "test@example.com");
  await page.fill('input[name="password"]', "password123");
  await page.click('button[type="submit"]');

  // Verify redirect to dashboard
  await expect(page).toHaveURL("http://localhost:3000/dashboard");

  // Verify dashboard content
  await expect(page.locator("h1")).toContainText("Welcome");
});
```

## 🎨 Tests de composants React

### React Testing Library

```javascript
import { render, screen, fireEvent } from "@testing-library/react";
import { Button } from "./Button";

describe("Button Component", () => {
  test("renders button with label", () => {
    render(<Button label="Click me" />);
    expect(screen.getByText("Click me")).toBeInTheDocument();
  });

  test("calls onClick when clicked", () => {
    const handleClick = jest.fn();
    render(<Button label="Click me" onClick={handleClick} />);

    fireEvent.click(screen.getByText("Click me"));

    expect(handleClick).toHaveBeenCalledTimes(1);
  });

  test("is disabled when disabled prop is true", () => {
    render(<Button label="Click me" disabled />);
    expect(screen.getByText("Click me")).toBeDisabled();
  });

  test("applies correct variant class", () => {
    render(<Button label="Click me" variant="primary" />);
    const button = screen.getByText("Click me");
    expect(button).toHaveClass("btn--primary");
  });
});
```

## 📏 Couverture de code

### Objectifs raisonnables

- **Statements** : 80%+
- **Branches** : 75%+
- **Functions** : 80%+
- **Lines** : 80%+

### ⚠️ Attention

> La couverture à 100% ne garantit pas l'absence de bugs !
>
> Mieux vaut 50% de tests pertinents que 100% de tests inutiles.

## ✅ Checklist pour de bons tests

### Structure

- [ ] Tests organisés avec describe/test
- [ ] Noms descriptifs et clairs
- [ ] Pattern AAA appliqué
- [ ] Tests indépendants

### Qualité

- [ ] Testent le comportement, pas l'implémentation
- [ ] Couvrent les cas normaux et edge cases
- [ ] Utilisent des mocks appropriés
- [ ] Rapides à exécuter

### Maintenance

- [ ] Faciles à comprendre
- [ ] Faciles à modifier
- [ ] Documentation si nécessaire

## 💡 TDD (Test-Driven Development)

### Le cycle Red-Green-Refactor

```
1. 🔴 RED : Écrire un test qui échoue
2. 🟢 GREEN : Écrire le minimum de code pour passer le test
3. 🔵 REFACTOR : Améliorer le code sans changer le comportement
4. Répéter
```

### Exemple TDD

```javascript
// 1. 🔴 RED - Test qui échoue
test("calculates average of array", () => {
  expect(average([1, 2, 3])).toBe(2);
});

// 2. 🟢 GREEN - Implémentation minimale
function average(numbers) {
  return numbers.reduce((a, b) => a + b) / numbers.length;
}

// 3. 🔵 REFACTOR - Améliorer (gérer edge cases)
function average(numbers) {
  if (!numbers || numbers.length === 0) {
    return 0;
  }
  return numbers.reduce((a, b) => a + b, 0) / numbers.length;
}

// Ajouter test pour le edge case
test("returns 0 for empty array", () => {
  expect(average([])).toBe(0);
});
```
