# Software Craftsmanship

Le software craftsmanship met l'accent sur la qualité du code, l'apprentissage continu et la responsabilité technique.

## Ressources et métaphores

### Stone Soup

- Partir d'un noyau simple
- Construire progressivement avec les contributions du groupe
- Favoriser l'engagement autour d'un premier résultat concret

### Frog Soup

- Les petits compromis répétés peuvent abîmer un système sans signal d'alarme immédiat
- Il faut garder une vigilance active sur la dette technique

## Ce que ça implique en pratique

- Écrire du code lisible
- Chercher les boucles de feedback courtes
- Automatiser la validation
- Accepter la rigueur dans la spec et les tests
