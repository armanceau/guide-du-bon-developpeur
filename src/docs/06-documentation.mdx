# 📖 Documentation

Une bonne documentation rend un projet plus simple à utiliser, à maintenir et à transmettre.

## Pourquoi documenter ?

### Pour les autres

- Facilite l'onboarding
- Rend le code plus compréhensible
- Réduit le support informel

### Pour vous

- Vous oublierez comment le code fonctionne
- La documentation aide à clarifier les idées
- Elle évite de redécouvrir les mêmes règles métier

### Pour le projet

- Réduit la dépendance aux individus
- Facilite la maintenance
- Améliore la qualité globale

## Types de documentation

### README

Le README doit permettre de démarrer le projet sans devoir deviner les étapes.

### Documentation API

- JSDoc pour les projets JavaScript/TypeScript
- Annotations et exemples pour les API REST
- Swagger ou OpenAPI pour exposer clairement les contrats

### Documentation de code

- Les types aident déjà à documenter
- Les commentaires doivent expliquer le pourquoi, pas répéter le code
- Un code explicite vaut mieux qu'un long commentaire périssable

## Bonnes pratiques

- Garder les exemples courts et à jour
- Documenter les conventions d'équipe
- Mettre à jour la doc en même temps que le code
- Supprimer la documentation devenue obsolète

## Ce qu'il faut éviter

- Documenter ce qui est déjà évident dans le code
- Laisser une doc obsolète plus de quelques sprints
- Remplacer une mauvaise structure de code par des commentaires
