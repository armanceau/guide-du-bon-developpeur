# Paradigmes

Un paradigme de programmation est un modèle conceptuel qui guide la manière de penser, structurer et écrire un programme.

## Domain-Driven Design

- Met le métier au centre
- Réduit l'écart entre dev et experts métier avec un ubiquitous language
- Utilise des bounded contexts pour limiter la complexité
- Rend le code plus fidèle aux règles métier

### Exemple bancaire

- Entités: Compte, Transaction, Client
- Règle métier: un solde ne doit jamais devenir négatif sur un compte épargne

## Test-Driven Development

- Écrire les tests avant le code
- Cycle Red, Green, Refactor
- Renforce la qualité et limite les régressions

### Démarche

1. Écrire un test qui échoue
2. Écrire le code minimal pour le faire passer
3. Refactorer sans casser le comportement

## Behavior-Driven Development

- Extension du TDD orientée comportement attendu
- Utilise souvent Given, When, Then
- Rend les critères d'acceptation compréhensibles par toute l'équipe

## Relation entre les trois

- DDD aide à structurer le métier
- TDD sécurise l'implémentation
- BDD clarifie les comportements attendus
